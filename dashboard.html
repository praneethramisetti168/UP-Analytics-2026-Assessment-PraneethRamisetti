📋 Methodology
End-to-end pipeline from raw sensor data to deployed prediction service
Preprocessing Pipeline
1

Missing Value Parsing

Parsed "na" strings as NaN. Dropped 13 columns with >70% missingness. Applied median imputation on remaining columns — robust to outliers and skew.

2

Missingness Indicators

Added binary flags for columns with 20–70% missing data. Encodes the "was this sensor reading available?" signal as an additional feature.

3

Row-Level Aggregates

Added per-row: null count, null fraction, mean, std, min, max across all sensors — capturing overall sensor health at the observation level.

4

Feature Scaling

StandardScaler applied after imputation for models sensitive to feature scale (Logistic Regression). Tree models operate on raw values.

5

Threshold Optimisation

Classification threshold selected by maximising F1 on the validation PR curve. Final threshold: 0.7147, reflecting the 10–50× cost asymmetry of missed failures.

Assumptions & Limitations
NO TIMESTAMP
Time-based split approximated using row ordering. If rows are shuffled in source, this may not hold.
ANONYMIZED FEATURES
Sensor names (e.g., aa_000, cr_000) are anonymized — domain interpretation requires Scania's data dictionary.
STATIC MODEL
Trained offline. Real-time streaming requires an additional data pipeline and model versioning.
NO SMOTE
To avoid data leakage, SMOTE was not used. Imbalance handled via scale_pos_weight instead.
